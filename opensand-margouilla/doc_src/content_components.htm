Components

The Open Network Toolkit is composed by:
- A C++ runtime that provides platform independant messaging, threads management, timers, traces...
- A set of common protocols ready to use (IP/ATM/Ethernet layers...).
- A Project library that handles XML files and code generation. This library is used by the graphical editor.
- A graphical editor with SDL design style.

The Runtime

The runtime provides support for real time and compressed time events:
- Timers
- Thread management
- Messages with C structures
- Thread-safe message boxes
- Multicast channels
- IPv4 IPv6 unicast and multicast socket event
- Delay and bandwidth limitations
- Flow chart traces
- Measurements on links use
- State machine states
- Remote debugging
- Usable with SDL generated code
- Addressing thanks Id
- Message routing thanks links and ports
- Protocol stacks linked with upper/lower layers
- ...

The runtime is coded in C++.

The Common Blocs

Common blocs implements common functionnlaities such as:
- Message forwarders
- Leaky bucket
- UDP layer
- IP layer with routing table
- Ethernet layer interface
- ATM cells interface
- Interception and insertion of ethernet packet over the wire
- ...

The Project Library

This library handles XML files describing:
- A whole system
- Processes interfaces and state machines
- Messages used
- Mapping between blocs and blocs managers
- Code generation for use with the runtime.

This library is used by the graphical editor.

The Graphical Editor

The graphical editor provides developpers with visual design of C++ bloc states and events following the graphical SDL standard.
Messages are exchanged between blocs by using unicast unidirectionnal links, or multicast broadcast channel. Those links are characterised by delay, and bandwidth, that allows you to simulate radio links between a mobile phone and its base station, an usb bus, or a satellite hub stations and its terminals receiving broadcasted packets.

The editor uses an open XML format for its data files and generates the code for the C++ classes. A target manager allows to transparently distribute processes over a network without coding any socket function... Marshaller are automatically generated for C messages structures (no more need to worry about padding or little endian). The links and channels can be mapped on unicast and multicast IPV4 or IPv6 links. This allows you to graphically define and validate a multicast protocol with one client and one server, compile a client and a server process, then run one server and many client process.
